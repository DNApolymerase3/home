Propositional Logic

tagged with rust web development typescript

Published on September 20, 2024

In my recent endeavors with static compilation, I've encountered some fascinating challenges regarding truth tables and propositional logic. This post details my journey through implementing a basic propositional logic system in Rust, and the unexpected complexities that arose.

The Initial Approach

I started with a simple representation of boolean expressions: 

enum Expr {
    Var(String),
    And(Box, Box),
    Or(Box, Box),
    Not(Box),
}

This seemed straightforward at first, but complications arose when dealing with variable binding and evaluation. The static nature of Rust's type system both helped and hindered here - while it prevented many runtime errors, it also made certain dynamic operations more challenging.

Truth Tables and Evaluation

Generating truth tables proved more complex than anticipated. Here's a simplified version of the evaluation function: 

fn evaluate(expr: &Expr, bindings: &HashMap) -> Result {
    match expr {
        Expr::Var(name) => bindings.get(name)
            .ok_or(Error::UnboundVariable(name.clone())),
        Expr::And(left, right) => Ok(
            evaluate(left, bindings)? && evaluate(right, bindings)?
        ),
        Expr::Or(left, right) => Ok(
            evaluate(left, bindings)? || evaluate(right, bindings)?
        ),
        Expr::Not(expr) => Ok(!evaluate(expr, bindings)?)
    }
}

Lessons Learned

Through this implementation, I learned several valuable lessons about static compilation:

  • Type systems can enforce correctness, but sometimes at the cost of flexibility
  • Recursive structures require careful consideration of memory layout
  • Error handling in recursive operations needs special attention

Future Improvements

Moving forward, I plan to:

  • Implement more efficient truth table generation
  • Add support for more complex logical operations
  • Explore possibilities for parallel evaluation

This project has been a fascinating dive into the intersection of logic and systems programming. While the challenges were significant, they've led to a deeper understanding of both propositional logic and Rust's type system.
